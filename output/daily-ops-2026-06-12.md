### 2. Wrote research brief — `author-authority-for-saas-content`

**File**: `research/brief-author-authority-for-saas-content-2026-06-12.md`  
**Primary keyword**: author authority SEO (~500/mo, KD: ~32)  
**Cluster**: Cluster 8 — E-E-A-T and Authority Building (Supporting #2)

**Five competitor gaps identified:**
1. No distinction between author bios (strings) and author entities (Knowledge Graph nodes) — every existing guide conflates these, missing the entity compounding mechanism
2. No author model framework — no guide addresses the SME-as-author vs. writer-as-expert vs. ghostwritten executive decision, with E-E-A-T tradeoffs per model
3. No technical implementation guide for Person schema in SaaS context — `sameAs` properties never explained, Yoast/RankMath defaults insufficient for entity building
4. No measurement framework — no competitor explains how to know if author authority is moving rankings (entity recognition signals, branded author search, rich results rate, cluster performance by author)
5. No author authority decay and succession protocol — when a key author leaves, no guide covers the transition: attribution update, entity continuity, institutional author alternatives

**Differentiating angle**: Author authority is not bio writing — it's entity building. Google's Knowledge Graph distinguishes between an author as a string (text in your CMS) and an author as an entity (independently verifiable node with consistent signals across multiple surfaces). The entity compounding effect is the strategic opportunity: each new article strengthens the entity, and the entity in turn strengthens every existing piece. Most B2B SaaS companies have published under anonymous or rotating bylines — zero entity signal accumulated despite high content volume. Post-HCU, this is now a competitive gap.
