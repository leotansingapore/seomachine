## Technical note: content_scrubber.py bug

The `scrub()` method in `data_sources/modules/content_scrubber.py` emptied the file when called after `Write`. The file was restored manually. The scrubber returns a `(str, Dict)` tuple -- the write must use `cleaned, stats = scrubber.scrub(content)` pattern. The bug appears when the scrubber is invoked in a context where the `textstat` dependency chain causes partial module initialization. Recommend investigating before relying on the scrubber in automated pipelines.

**Blocker**: `data_sources/modules/content_scorer.py` also fails -- missing `textstat` pip package. Install with: `pip install textstat`
