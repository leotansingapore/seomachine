### 2. Wrote research brief — `saas-onboarding-best-practices`

**File**: `research/brief-saas-onboarding-best-practices-2026-06-22.md`  
**Primary keyword**: SaaS onboarding best practices (~600/mo, KD: ~40)  
**Cluster**: Cluster 10 — SaaS Onboarding Optimization (PILLAR)

**Five competitor gaps identified:**
1. Onboarding defined only as the first-login flow — competitors treat onboarding as the welcome modal, product tour, and checklist; none map the full arc (sign-up → activation → habit formation → power user → expansion). Teams optimize the wrong stage.
2. No PLG vs. SLG architecture separation — PLG (self-serve, product-first, event-driven) and SLG (CSM-led, milestone-driven, high-touch) are fundamentally different architectures. Most guides conflate them without the decision logic for which to build.
3. No activation framework — competitors list tactics (add tooltips, write a welcome email). None provide the structural process: identify aha moment → define signal events → build minimum activation sequence → measure activation rate → iterate.
4. No activation rate measurement guidance — "activation rate" appears in most guides but is never defined precisely enough to act on. No guide explains the signal event approach, how to identify it from cohort data, or what a healthy benchmark looks like by product type.
5. No onboarding audit methodology — all guides assume you're designing from scratch. No diagnostic framework for teams with existing onboarding: where to look, how to classify failure modes, how to prioritize fixes by leverage.

**Differentiating angle**: SaaS onboarding is not a welcome sequence — it's a value architecture problem. Most products deliver features instead of outcomes during onboarding. The framework: identify aha moment → design minimum activation sequence → build value ladder → measure with signal events.
