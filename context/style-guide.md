# SEO Agency Style Guide

This guide defines writing conventions, formatting standards, and editorial guidelines for all SEO agency content. Client-specific overrides live in `clients/<slug>/context/style-guide.md`.

## Instructions
Update this guide as agency standards evolve. Client overrides take precedence over these defaults.

---

## Grammar & Mechanics

### Capitalization

**Headlines & Subheadings**:
- Sentence case (Only capitalize first word and proper nouns)
- **Decision**: Sentence case across all content types

**Agency & Product Names**:
- Capitalize proper product and brand names exactly as the brand spells them
- Industry-standard tool names: Google Search Console, Ahrefs, SEMrush, etc.

**Industry Terms**:
- SEO: always caps
- SERP / SERPs: always caps
- URL / URLs: always caps
- CTA: always caps
- keyword: lowercase unless starting a sentence
- backlink: lowercase

### Numbers

**When to Spell Out**:
- Spell out: One through nine
- Use numerals: 10 and above
- Exceptions:
  - Percentages: Always use numerals (5%, 50%)
  - Money: Always use numerals ($5, $500)
  - Measurements: Always use numerals (5 GB, 50 MB)
  - Stats in lists: Always use numerals for scannability

**Large Numbers**:
- Use commas: 1,000+
- Spell out magnitude: 1 million, 2 billion
- Revenue/budgets: $5,000 or $5K (use $5K for informal, $5,000 for formal reports)

### Punctuation

**Oxford Comma**:
- Yes (A, B, and C)
- **Decision**: Always use Oxford comma

**Em Dashes**:
- Style: -- (double hyphen) or em dash with no surrounding spaces
- Usage: Parenthetical statements or emphasis
- Example: "Technical SEO--when done systematically--compounds over time."

**Quotation Marks**:
- Straight quotes for inline code, curly quotes for prose
- **Decision**: Curly quotes in editorial content

**Ellipses**:
- Three dots: ...
- No spaces before or after
- Use sparingly: only for omitted text in direct quotes

### Abbreviations & Acronyms

**First Use**:
- Spell out on first use, acronym in parentheses
- Example: "Google Search Console (GSC) shows impression data..."
- Exceptions for universally known: SEO, URL, CTA, API

**Common SEO Acronyms**:
- GSC: Google Search Console
- GA4: Google Analytics 4
- SERP: Search Engine Results Page
- CTR: Click-Through Rate
- DA: Domain Authority
- DR: Domain Rating
- CWV: Core Web Vitals
- E-E-A-T: Experience, Expertise, Authoritativeness, Trustworthiness

**Latin Abbreviations**:
- e.g. and i.e.: acceptable in parenthetical notes
- etc.: avoid -- be specific instead
- **Decision**: Prefer plain English in body copy

## Word Choice & Usage

### Preferred Terms

**Say This** -> **Not That**:
- organic traffic -> free traffic
- search visibility -> Google ranking (broader than just one engine)
- publish -> post (more professional)
- data shows -> studies suggest (only if you have actual data)
- increased by X% -> improved (use specific numbers)
- target keyword -> focus keyword
- content gap -> missing content
- search intent -> user intent
- technical audit -> website audit (be specific)

### Words to Avoid
- "Revolutionary", "game-changing", "cutting-edge" -- vague and overused
- "Very", "really", "actually" -- filler
- Passive voice when active is clearer ("we increased traffic" not "traffic was increased")
- "Click here" or "read more" -- use descriptive anchor text
- Superlatives without data: "best", "fastest", "most powerful"
- Jargon without explanation on first use

### Data-Driven Language
- Lead with numbers when you have them: "Rankings improved 34% in 90 days" not "Rankings improved significantly"
- Cite sources inline: "According to Ahrefs (2024), 96.55% of pages get zero organic traffic"
- Use ranges when exact figures aren't available: "typically 3-6 months"
- Distinguish correlation from causation

### Inclusive Language
- Use gender-neutral language ("they" instead of "he/she")
- Avoid idioms that may not translate globally
- Write for an international business audience unless client context specifies otherwise

## Formatting Standards

### Text Formatting

**Bold**:
- Use for: Key concepts, important takeaways, primary terms on first use
- Don't overuse: Max 2-3 instances per section
- Example: "**Core Web Vitals** measure real-world user experience."

**Italics**:
- Use for: Titles of tools/publications, emphasis (sparingly), technical terms being defined
- Example: "The concept of *search intent* refers to the goal behind a query."

**Underline**:
- Avoid (reserve for hyperlinks only)

**ALL CAPS**:
- Avoid except for standard acronyms (SEO, GSC, URL, CTA)

### Lists

**Bulleted Lists**:
- Use for: Non-sequential items, features, benefits, recommendations
- Capitalization: Sentence case
- Punctuation: Period if complete sentence, no period if fragment
- Parallel structure required (all sentences or all fragments)

**Numbered Lists**:
- Use for: Sequential steps, rankings, ordered priorities
- Format: 1. 2. 3. (period after number)
- Same capitalization and punctuation rules as bullets

**Nested Lists**:
- Maximum 2 levels deep
- Use different markers for sub-items

### Links

**Anchor Text**:
- Descriptive -- tell the reader what they're clicking to
- Keyword-rich when natural
- 2-6 words typically
- Correct: "read our technical SEO checklist"
- Incorrect: "click here"

**Link Formatting**:
- External links: open in new tab
- Internal links: same tab

### Code & Technical Elements

**Inline Code**:
- Use backticks for: URLs, file names, HTML tags, code snippets
- Example: "Add the canonical tag: `<link rel='canonical' href='...' />`"

**Code Blocks**:
- Use for: Multi-line code, schema markup examples, robots.txt examples
- Include language identifier for syntax highlighting

### Callout Boxes / Asides

**When to Use**:
- Key stats or findings worth highlighting
- Warnings or caveats
- Pro tips with specific tactical value

**Format**:
- Blockquote style for inline callouts
- Keep to 1-3 sentences

## Content Structure

### Article Introduction
**Standard Structure** (150-250 words):
1. **Hook** (1-2 sentences): Lead with a specific number, stat, or direct question
2. **Problem** (2-3 sentences): What challenge does this address?
3. **Promise** (2-3 sentences): What will the reader know or be able to do after reading?
4. **Credibility** (optional, 1-2 sentences): Data point or proof that establishes authority

**Keyword Placement**:
- Primary keyword within first 100 words
- Natural integration only -- no keyword stuffing

### Section Length
- **Minimum**: 150 words per section
- **Maximum**: 500 words per section (split into subsections if longer)
- **Ideal**: 250-350 words per main section

### Conclusion
**Standard Structure** (150-250 words):
1. **Recap** (3-5 bullets or short paragraph): Key takeaways
2. **Action** (1-2 sentences): What should the reader do next?
3. **CTA** (1-2 sentences): Call-to-action (audit, consultation, tool, related content)
4. **Forward-looking** (optional, 1 sentence): Final thought with momentum

## SEO-Specific Style

### Meta Titles
- 50-60 characters including spaces
- Include primary keyword
- No brand suffix unless client requires it
- No ending punctuation

### Meta Descriptions
- 150-160 characters including spaces
- Include primary keyword
- Include a clear benefit or call-to-action
- End with a complete thought

### URL Slugs
- Lowercase only
- Hyphens between words
- Include primary keyword
- 3-5 words ideal
- Format: `/blog/primary-keyword-phrase`

### Alt Text
- Describe what the image shows, not what it is
- Include keyword naturally only if genuinely relevant
- 125 characters or less
- Never start with "image of" or "photo of"

## Dates & Time

**Date Format**:
- Month DD, YYYY (January 15, 2025) in editorial content
- YYYY-MM-DD in data exports and filenames

**Time**:
- 12-hour format: 3:00 p.m.

**Time Zones**:
- Specify when relevant: "3:00 p.m. SGT"

## Statistics & Data

### Citing Sources
- Always cite statistics with sources
- Format: "According to [Source] ([Year]), [statistic]."
- Link to original source when possible
- Flag data older than 2 years -- verify or replace

### Presenting Numbers
- Round large numbers for readability: "1.2 million" not "1,234,567"
- Use % symbol: 15% (not "15 percent")
- Use $ for money: $500 (not "500 dollars")
- Include commas: 10,000 not 10000
- Use specific numbers over vague claims: "increased by 47%" not "significantly increased"

## Images & Media

### Image Captions
- Optional -- only include when the image needs context not covered by surrounding text
- Sentence case, end with period if complete sentence

### Screenshots
- Crop to show only the relevant portion
- Add arrows or highlights to direct attention
- Alt text should describe what is shown and why it matters

### Charts & Graphs
- Include when data visualization aids understanding more than a sentence would
- Keep design minimal -- label axes, include units
- Alt text should describe the key finding, not just "chart showing X"

## Brand-Specific Guidelines

### Agency References
- Refer to the agency by its name (fill in when onboarding)
- Use "we" for first-person agency content
- Use "you" to address the client or reader directly

### Client References
- Never name a client without explicit written approval
- Use anonymized descriptions: "a mid-size e-commerce brand" or "a B2B SaaS company"

### Competitor References
- Name competitors only when it serves the reader (e.g., comparison content)
- Be factual and fair -- no disparaging language
- Focus on differentiation, not criticism

## Accessibility

### Screen Reader Friendly
- Descriptive link text (not "click here")
- Alt text for all images
- Proper heading hierarchy (H1 -> H2 -> H3, no skipping levels)
- Never use color alone to convey meaning

### Plain Language
- Target 8th-10th grade reading level for general content
- Define technical SEO terms on first use
- Short sentences and paragraphs
- Break complex ideas into numbered or bulleted steps

## Voice & Tone

### Core Voice Characteristics
1. Direct and specific -- lead with data, skip the preamble
2. Authoritative but not condescending
3. Systems-focused -- explain the mechanism, not just the tactic
4. Results-oriented -- every recommendation connects to a business outcome
5. Honest about tradeoffs -- no silver bullets

### Tone Variations
- **How-to content**: Instructive, step-by-step, action-oriented
- **Strategic content**: Authoritative, analytical, outcome-focused
- **Industry analysis**: Insightful, evidence-based, forward-looking
- **Client-facing content**: Benefit-focused, clear, specific to their goals

- [ ] Uses preferred terminology

- [ ] Links are descriptive
- [ ] Voice and tone appropriate for content type

- [ ] Strong introduction (hook, problem, promise)
- [ ] Logical section flow
- [ ] Proper heading hierarchy (no skipped levels)
- [ ] Effective conclusion with CTA
- [ ] Appropriate section lengths

**SEO**:
- [ ] Primary keyword in first 100 words
- [ ] Meta title and description optimized
- [ ] Internal and external links included
- [ ] All images have alt text
- [ ] URL slug includes primary keyword

- [ ] All statistics cited with sources
- [ ] No claims older than 2 years without verification
- [ ] No broken links
- [ ] Provides specific, actionable value
- [ ] Ready for publication
